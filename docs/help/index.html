List of operatorsguide Operators are small character sequences that imply functions like add, subtract, etc. They can be binary (take two arguments) or unary (apply to one number). Binary operators are used like 3*4, which is casted by the program into mul(3,4), for example. The binary operators are:
+ - add (Addition)
- - sub (Subtraction)
* - mul (Multiplication)
/ - div (Division)
^ - pow (Exponentiation)
** - pow (Exponentiation)
= - equal
== - equal
!= - not_equal
> - gt (Greater than)
>= - gt_equal (Greater than or equal)
< - lt (Less than)
<= - lt_equal (Less than or equal)
Binary operators have an order of precedence, which is exponents first, then multiplication and division, then addition and subtraction, then the comparison operators. Example: 4*5+9 becomes add(mul(4,5),9) because multiplication has higher precedence. The prefix unary operators (goes before expression) are:
- - neg (Negative)
The suffix unary operators (goes after expression) are:
! - factorial. 4*5!+2 = add(mul(4,factorial(5)),2)
Unitsguidemeasurement Units store types of units of measurement. They must be referenced inside the [] square brackets (do not confuse them with the square brackets in accessor notation). An example of a unit is meters, which stores length. Six metric base units are supported, and the dollar and bit are additionally added. They are stored as a list of base units with their powers. For example area is stored as [m^2]. All units are stored in terms of metric base units, and some have support for metric prefixes. Units interact with certain operators, like with multiplication, their powers are added, with division the powers are subtracted, etc. However, with most functions, the units are left untouched. The base units are: m for Meter, kg for kilogram, s for second, A for Amp, K for Kelvin, mol for mole, b for bits, and $ for dollars. If the power ever goes above 127 or below -127, an overflow error will occur, and the powers can only be integers. See the list of units in the help pages. [m]/[s] = [m/s][J/s] = [W] See more: www.mathsisfun.com

Base conversionguide There are multiple ways that different bases can be parsed, however there is currently no way to print in a different base. The first is a prefix signal. Prefix signals start with a zero (so it is not parsed as a variable), followed by a character, then the number. The supported prefixes are: 0b binary (2), 0t ternary (3), 0o octal (8), 0d decimal (10), and 0x hexadecimal (16). Note that the exponent character is also parsed in that base, so 0b11e101 is 3 * 2^5. The other way to parse bases is using the square bracket underscore notation. This looks something like [0A1]_16. The base is the number after the underscore, which can be computed, but it must be constant. Note that, like in this example, all numbers must start with a number from 0-9, so hexadecimals must start with 0. The maximum base is 36 and uses the characters 0-9 and then A-Z for 10-36 (characters must be uppercase). The minimum base is base 2. ob101 = 50b11e101 = 3 * w^50xA1 = 0xA*16 + 0x1 = 1610xA1 = [0A1]_16[15]_15 = 20 See more: en.wikipedia.org
Accessor notationguideget, index Accessor notation is a quick alias to the function get. It is written as square brackets following a parenthesis block, string, vector, map or function call. For vectors, it returns the value at that index, with the first value having index 0. For a map, it returns the value that is paired with that key. For a string, it returns the character code for the character at that index, with the first character having index 0. If you would instead like to use square brackets for naming units, precede it with an operator (usually multiplication) so it is not treated as an accessor. <1,2,3>[0] = 1<1,2,3>[1] = 2{4:3,"4":2}[4] = 3"Hello World"[1] = "e"

It also allows accessor notation in assignment to allow the changing of a single memeber in a vector, map, or string, accessors can be nested in this form. Commandguide Commands are functions that can access special parts of the program. Commands are prefixed by the '/' character. Commands interact with top-level features that can't, or wouldn't make sense to, be in a function. A list of commands is available in the help pages.
Historyguideanswer, memory, previous The history is a record of all previous calculations. It can be accessed in several ways: first, the ans variable will always return the previous answer when presented with no arguments. Second, the ans function will return that element in the history, e.g. ans(5) will return the fifth element, and ans(-2) will return the one that is two before the previous calculation. ans and ans(-1) are equivalent (the previous element). Finally, you can also use the $1 number that is presented whenever the answer was given. ans = ans(-1)$3 = ans(3)
Variablesguidevalues, custom Variables are global objects that store a value under a string. They can be created with the assignment operator, and can be accessed with their name. Example: a=10 will set the variable a to 10, whenever a is referenced from now on, it will evaluate to 10. Variables can also be used as custom functions, for example g(a,b)=a*b will create a function g that returns the first argument multiplied by the second argument. This is internally stored as a lambda function, which is essentially an expression with arguments encapsulated in a value.
Typesguide Each value in the program can be stored in one of 6 different types:
num - number (a complex float)
arb - arb (an arbitrary precision float)
vec - vector (stores a list of values)
lmb - lambda (encapsulates expressions into mini functions)

These types allow for many different features and are widely supported in many of the different functions.
Numbernumtypefloating, double The number class has three components: a real decimal, and imaginary decimal, and a unit. These can be treated as a single package. Each of the two decimals is stored to 15 digits of precision. More precision can be stored in the arb type. The typeof function returns 0 for this type. 2-8i2.5+4i*[m]2[kg]inf See more: en.wikipedia.org
Arbarbtypep, precision, arbitrary, float The arb type is very similar to the number type, except that the decimals are stored with arbitrary precision, and it does not have imaginary numbers. To cast from arb to a number, use the tonumber function. To cast from number to arb, use the toarb function, and precision can be specified in the second argument. They can also be created with the 'p' symbol in a decimal literal. If the number of digits is more than 15 it will also store it with extra precision as in the previous example. When two arb types (or an arb and a number) are put together in a binary operation, the return value will have the highest precision of both. The 'p' symbol must come after the 'e' symbol for exponents. On the terminal interface version, arb may not be supported if it is compiled without it. The typeof function returns 1 for this type. 1.5p20 * 2p5 = 3.0p201.5p20 = 1.50000000000000000001.5e2p50 = 150p50 See more: github.com
Vectorvectypelist, array, iterator, index The vector type stores an arbitrarily long list of values. The elements can be of any type. They are created with the angle brackets < and > as is commonly used in linear algebra. There are many different built in functions that handle vectors, and the complete list can be seen in the list of functions help page. Some common ones are get, fill, map_vector, length, magnitude, normalize, and sort. The most common function is get, which is also aliased by accessor notation. Vectors can be used to store Euclidian point positions, return multiple values from a function, or store a list. Vectors apply to many common functions like sqrt, exp, and every other elementary operation. See the pages for each function to see if they apply to vectors. Lambdalmbtypeanonymous function, => The lambda type stores a nameless function that can have any number of arguments, and a single output. They use arrow notation similar to JavaScript. Example: x=>x+1 should be read as "x goes to x plus one". For zero argument lambdas, use an underscore character _=>rand, for more than one, use a comma separated list enclosed in parenthesis (x,t)=>x*t. As long as they follow the variable naming conventions. Lambdas can also be nested and are dynamic types. There are many different functions that take advantage of lambdas, common ones are: run, apply, sum, product, fill, map, and sort. Variables that store anonymous functions can be treated as regular functions with that name like a(x)=x^2. Stringstrtypecharacter, text, words The string type stores text (as a list of characters). Strings can be used to print information, create dynamic evaluations, or throw errors. They are enclosed by the double quotes " (no other wrapping is available). There are multiple different string methods, like: eval, substr, lowercase, uppercase, error, print, indexof, and replace. In order to prevent character conflict, double quotes within strings must be preceded by the \ backslash character, backslashes must be escaped by another backslash. Mapmaptyperelation, dictionary, object, key The map type stores key-value mappings, it is represented using the {} curly bracket syntax. Each pair is separeted by commas and the elements in the pair are separeted by the colon. Example: {1:4,"yes":8} means 1 maps to 4, and "yes" maps to 8. Any value (even lambdas) can be either a key or value. Maps are not used much in builtin functions, but they are extremely useful in storing data in a logical way, or to create an object oriented design. The map is internally stored as a binary search tree, however the different value types can obviously not be compared, so the ordering is more approximate. Functions that support the map are: get, concat, and length. get is aliased by accessor notation similar to JavaScript. *** Algebra:

solvequad(a,b,c) - Solve Quadratic. Returns two solutions for a quadratic of the form ax^2+bx+c.
Compiles to: solvequad=(a,b,c)=>(-b- <-1,1>*sqrt(b^2-4a*c))/2a


*** Complex:

comp_cis(angle) - Cosine i sine. Returns the complex number with that angle and a magnitude of one. This is often called "cosine i sine" becasue it is equal to cos(angle)+i*sin(angle).
Compiles to: comp_cis=(angle)=>cos(angle)+i*sin(angle)

comp_complement(z) - Complex complement. Returns the complement of a complex number, the complement of a+bi is a-bi.
Compiles to: comp_complement=(z)=>getr(z)-geti(z)

comp_magnitude(z) - Complex magnitude. Returns the magnitude of a complex number. For z=a+b*i, comp_magnitude(z) returns sqrt(a^2+b^2).
Compiles to: comp_magnitude=(z)=>sqrt(getr(z)^2+geti(z)^2)


*** Random:

rand_int(min,max) - Random integer. Returns a random integer in the interval [min,max).
Compiles to: rand_int=(min,max)=>floor(rand()*(max-min)+min)

rand_member(data) - Random member. Returns a random member from a vector.
Compiles to: rand_member=(data)=>get(data,floor(rand*length(data)))

rand_range(min,max) - Random within range. Returns a random number in the interval [min,max).
Compiles to: rand_range=(min,max)=>rand()*(max-min)+min


*** Statistics:

binomial(p,n,x) - Binomial distribution. Returns the height of the binomial distribution with probability of success p, and x successes out of n trials.
Compiles to: binomial=(p,n,x)=>ncr(n,x)*p^x*(1-p)^(n-x)

correlation(datax,datay) - Correlation coefficient. Returns the correlation coefficient of two data sets. The two data sets are paired data, meaning each index matches them up.
Compiles to: correlation=(datax,datay)=>covariance(datax,datay)/stddev(datax)/stddev(datay)

mean(data) - Mean. Returns the mean of data. This is the sum of all of the members divided by the length.
Compiles to: mean=(data)=>sum(data)/length(data)

mean_geom(data) - Geometric mean. Returns the geometric mean of data. This is every member multiplied together, then the root with the index of the number of elements is taken
Compiles to: mean_geom=(data)=>product(data)^(1/length(data))

ncr(n,r) - Choose. Returns the number of combinations of size r from a set of size n. A combination is an unordered permutation.
Compiles to: ncr=(n,r)=>factorial(n)/factorial(n-r)/factorial(r)

npr(n,r) - Permutations. Returns the number of permutations of size r from a set of size n. A permutation is an ordered combination.
Compiles to: npr=(n,r)=>factorial(n)/factorial(n-r)

p(event,count) - Probability. Runs an event count times and adds up all of the results, it then divides by count. Example: p(_=>(rand>0.5),1000) = 0.5.
Compiles to: p=(event,count)=>sum(_=>event()!=0,0,count)/count

pop_stddev(data) - Population standard deviation. Returns the population standard deviation of a data set.
Compiles to: pop_stddev=(data)=>stddev(data)*sqrt(length(data)-1)/sqrt(length(data))

prob_complement(p) - Probability complement. Returns the complement of p. This is equal to 1-p.
Compiles to: prob_complement=(p)=>1-p

stddev(data) - Standard deviation. Returns the sample standard devation of a data set.
Compiles to: stddev=(data)=>run(u => sqrt(sum(x => (data[x] - u) ^ 2, 0, length(data)-1) / (length(data)-1)), mean(data))

z(x) - Z table. Returns the area to the left of the z-score x. Example: z(0) = 0.5.
Compiles to: z=(x)=>(erf(x/sqrt(2))+1)/2


*** Vector:

angle_between(u,v) - Angle between vector. Returns the angle between two vectors as a radian. It is equal to the arccosine of the dot product divided by the magnitude of both vectors.
Compiles to: angle_between=(u,v)=>acos(dot(u,v)/magnitude(u)/magnitude(v))

cross(u,v) - Cross product. Returns the cross product of two three dimensional vectors.
Compiles to: cross=(u,v)=>determinant(<<<1,0,0>,get(u,0),get(v,0)>,<<0,1,0>,get(u,1),get(v,2)>,<<0,0,1>,get(u,2),get(v,2)>>)

dot(u,v) - Dot product. Returns the dot product of two vectors. Assumes the maximum length of either vector.
Compiles to: dot=(u,v)=>sum(x=>get(u,x)*get(v,x),0,max(length(u),length(v)))
